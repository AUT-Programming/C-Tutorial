# تعریف الگوریتم

الگوریتم به مجموعه‌ای از گام‌های منطقی و مشخص برای حل یک مسئله یا انجام یک وظیفه گفته می‌شود که باید به ترتیب خاصی انجام شود. الگوریتم یک برنامه کامپیوتری نیست، بلکه طرح کلی برای چگونگی حل یک مشکل است که به صورت مستقل از زبان برنامه‌نویسی تعریف می‌شود. هدف از الگوریتم این است که روشی بهینه و موثر برای حل مسئله ارائه دهد.

#### ویژگی‌های اصلی یک الگوریتم

1. ورودی: الگوریتم باید حداقل یک ورودی داشته باشد.

2. خروجی: الگوریتم باید حداقل یک خروجی داشته باشد.

3. دقیق و بدون ابهام بودن: هر گام از الگوریتم باید واضح و بدون ابهام باشد.

4. محدودیت: الگوریتم باید پس از تعداد محدودی گام خاتمه یابد.

---

مثال ساده: الگوریتم یافتن بزرگترین عدد در یک لیست

1. اولین عدد لیست را به عنوان بزرگترین عدد در نظر بگیر.

2. عدد دوم را با بزرگترین عدد مقایسه کن. اگر بزرگتر بود، آن را به عنوان بزرگترین عدد جدید در نظر بگیر.

3. این کار را برای تمام اعداد لیست ادامه بده.

4. پس از بررسی تمام اعداد، بزرگترین عدد به دست خواهد آمد.

## تفاوت بین الگوریتم، سودوکد و زبان برنامه‌نویسی

+  الگوریتم: توصیف گام به گام از چگونگی حل یک مسئله است که به صورت مستقل از زبان برنامه‌نویسی و بدون هیچ قاعده نحوی خاص نوشته می‌شود.

+  سودوکد: نسخه‌ای نیمه‌رسمی از الگوریتم است که به زبان ساده و تقریباً شبیه به زبان برنامه‌نویسی نوشته می‌شود، اما نیازی به رعایت دقیق قواعد یک زبان خاص ندارد. سودوکد کمک می‌کند تا ساختار الگوریتم به صورتی که شبیه به کد نهایی باشد، بیان شود، در حالی که هنوز خوانایی بالایی دارد.

سودوکد برای یافتن بزرگترین عدد:

```

Start

Set max = first element of the list

For each element in the list:

    If element > max:

        Set max = element

End For

Print max

End

```

+ زبان برنامه‌نویسی: زبان برنامه‌نویسی برای پیاده‌سازی الگوریتم‌ها استفاده می‌شود. این زبان‌ها دارای قواعد و دستورات خاصی هستند که باید برای اجرای صحیح رعایت شوند.

####  تفاوت کلیدی

+ الگوریتم: طرح کلی و توصیف گام به گام از حل یک مسئله است، بدون اینکه به یک زبان خاص وابسته باشد.

+ سودوکد: نسخه ساده‌تر و شبه‌کدی از الگوریتم است که شبیه به زبان برنامه‌نویسی نوشته می‌شود.

+ زبان برنامه‌نویسی: پیاده‌سازی عملی الگوریتم با استفاده از یک زبان خاص است که کامپیوتر می‌تواند آن را اجرا کند.

# حل یک تمرین الگوریتمی

مسئله: محاسبه مجموع اعداد از 1 تا n.

هدف: نوشتن الگوریتمی که مجموع تمام اعداد صحیح از 1 تا یک عدد n را محاسبه کند.

#### طراحی الگوریتم:

1. ورودی: یک عدد صحیح n.

2. مقداردهی اولیه: مجموع را برابر 0 قرار می‌دهیم.

3. از 1 تا n، هر عدد را به مجموع اضافه می‌کنیم.

4. خروجی: مجموع نهایی را چاپ می‌کنیم.

## مبناهای عددی: دودویی، هگزادسیمال، اکتال، دهدهی و تبدیل بین آن‌ها

در این درس، ما به بررسی مبناهای مختلف عددی پرداخته و نحوه تبدیل بین آن‌ها را شرح می‌دهیم. همچنین به علامت‌گذاری اعداد، جمع کردن اعداد در مبناهای مختلف، و نمایش اعداد اعشاری (floating point) می‌پردازیم.

### ۱. مبناهای عددی

مبنا یا **پایه‌ی عددی**، تعداد رقم‌هایی است که در سیستم عددی استفاده می‌شود. در سیستم دهدهی (decimal) که به طور گسترده استفاده می‌شود، مبنای ۱۰ داریم که از ارقام ۰ تا ۹ تشکیل شده است. با این حال، سیستم‌های دیگری مانند دودویی (binary)، هگزادسیمال (hexadecimal) و اکتال (octal) نیز وجود دارند.

#### الف) سیستم دهدهی (Decimal)

سیستم دهدهی که رایج‌ترین سیستم عددی است، از مبنای ۱۰ استفاده می‌کند. هر عدد دهدهی ترکیبی از اعداد ۰ تا ۹ است.

**مثال**:

عدد `375` در مبنای ۱۰ به صورت زیر نمایش داده می‌شود:

```

375 = (3 × 10^2) + (7 × 10^1) + (5 × 10^0)

```

#### ب) سیستم دودویی (Binary)

سیستم دودویی از دو رقم ۰ و ۱ استفاده می‌کند و مبنای آن ۲ است. این سیستم در کامپیوترها و مدارهای دیجیتال استفاده می‌شود.

**مثال**:

عدد `1011` در مبنای ۲ به صورت زیر محاسبه می‌شود:

```

1011 (binary) = (1 × 2^3) + (0 × 2^2) + (1 × 2^1) + (1 × 2^0)

    = 8 + 0 + 2 + 1

    = 11

```

#### ج) سیستم اکتال (Octal)

سیستم اکتال دارای مبنای ۸ است و از ارقام ۰ تا ۷ استفاده می‌کند. این سیستم معمولاً در برنامه‌نویسی و سیستم‌های کامپیوتری استفاده می‌شود.

**مثال**:

عدد `157` در مبنای ۸ به صورت زیر است:

```

157 (octal) = (1 × 8^2) + (5 × 8^1) + (7 × 8^0)

    = 64 + 40 + 7

    = 111

```

#### د) سیستم هگزادسیمال (Hexadecimal)

در سیستم هگزادسیمال، مبنای ۱۶ داریم و از ارقام ۰ تا ۹ و حروف A تا F برای نمایش مقادیر استفاده می‌شود (A=10، B=11، ... F=15). این سیستم در برنامه‌نویسی برای نمایش مقادیر دودویی طولانی بسیار رایج است.

**مثال**:

عدد `2A3` در مبنای ۱۶:

```

2A3 (Hex) = (2 × 16^2) + (10 × 16^1) + (3 × 16^0)

    = 512 + 160 + 3

    = 675

```

### ۲. تبدیل بین مبناهای مختلف

#### الف) تبدیل دودویی به دهدهی

برای تبدیل یک عدد دودویی به دهدهی، کافی است ارقام عدد دودویی را به عنوان ضرایبی از توان‌های ۲ محاسبه کنید.

**مثال**: تبدیل `1101` (مبنای ۲) به دهدهی:

```

1101 (binary) = (1 × 2^3) + (1 × 2^2) + (0 × 2^1) + (1 × 2^0)

    = 8 + 4 + 0 + 1

    = 13  

```

#### ب) تبدیل دهدهی به دودویی

برای تبدیل یک عدد دهدهی به دودویی، عدد دهدهی را به ۲ تقسیم کنید و باقی‌مانده‌ها را یادداشت کنید تا به ۰ برسید.

**مثال**: تبدیل `19` به مبنای ۲:

```

19 ÷ 2 = 9  باقی‌مانده 1

9 ÷ 2  = 4  باقی‌مانده 1

4 ÷ 2  = 2  باقی‌مانده 0

2 ÷ 2  = 1  باقی‌مانده 0

1 ÷ 2  = 0  باقی‌مانده 1

10011 (binary)

```

#### ج) تبدیل هگزادسیمال به دودویی

هر رقم در سیستم هگزادسیمال معادل ۴ بیت (یا ۴ رقم دودویی) است.

**مثال**: تبدیل `2F` (مبنای ۱۶) به دودویی:

```
2 = 0010

F = 1111

00101111 (binary)

```

#### د) تبدیل دودویی به هگزادسیمال

برای تبدیل دودویی به هگزادسیمال، عدد دودویی را به گروه‌های ۴ بیتی تقسیم کنید و هر گروه را به هگزادسیمال تبدیل کنید.

**مثال**: تبدیل `10111001` به هگزادسیمال:

```

1011 = B

1001 = 9

پس عدد هگزادسیمال: B9

```

### ۳. جمع کردن اعداد در مبناهای مختلف

#### جمع دودویی

جمع کردن در مبنای ۲ قوانین خاصی دارد:

- 0 + 0 = 0

- 0 + 1 = 1

- 1 + 1 = 0 (و ۱ به ستون بعدی منتقل می‌شود)

**مثال**:

```

  1011

+ 1101

-------

 11000

```

#### جمع هگزادسیمال

جمع هگزادسیمال مانند جمع دهدهی است، اما با توجه به اینکه مبنای ۱۶ داریم، باید به جای ۱۰ تا ۹ از A تا F استفاده کنیم.

**مثال**:

```

  2A3

+  B7

------

  35A

```

### ۴. نمایش اعداد با علامت (Sign Bit)

در سیستم‌های کامپیوتری، برای نمایش اعداد منفی، از بیت علامت استفاده می‌شود. در سیستم دودویی با علامت، عددی که با ۰ شروع می‌شود مثبت و عددی که با ۱ شروع می‌شود منفی است. یکی از روش‌های نمایش اعداد منفی، استفاده از **متمم دو** (Two's Complement) است.

#### متمم دو

برای به دست آوردن متمم دو یک عدد:

1. تمام بیت‌های عدد را معکوس کنید (متمم یک).

2. ۱ واحد به نتیجه اضافه کنید.

**مثال**:

عدد `5` در دودویی: `00000101`

متمم دو برای `-5`:

```

1. معکوس کردن: 11111010

2. اضافه کردن 1: 11111011

```

### ۵. نمایش اعشاری (Floating Point Representation)

نمایش اعشاری یکی از روش‌های ذخیره‌سازی اعداد حقیقی در رایانه‌ها است که اجازه می‌دهد اعداد بسیار بزرگ یا بسیار کوچک به طور مؤثر ذخیره شوند. این روش معمولاً با استفاده از **فرمت IEEE 754** پیاده‌سازی می‌شود که شامل قسمت‌هایی برای علامت، نمایه و مقدار کسری است.

#### ساختار نمایش اعشاری

یک عدد اعشاری در فرمت اعشاری به صورت زیر تقسیم‌بندی می‌شود:

```

S | EEEEEEEE | MMMMMMMMMMMMMMMMMMMMMMM

```

- **S (علامت)**: یک بیت برای نشان دادن مثبت یا منفی بودن عدد. اگر S برابر با ۰ باشد، عدد مثبت است و اگر برابر با ۱ باشد، عدد منفی است.

- **E (نمایه)**: این بخش برای نشان دادن توان عدد ۲ استفاده می‌شود. معمولاً به صورت یک عدد صحیح بی‌نشان (unsigned integer) ذخیره می‌شود و برای اعداد منفی از یک **offset** (بازه) استفاده می‌شود.

- **M (مقدار کسری)**: این بخش شامل بخش کسری عدد است که به صورت نرمال‌شده (normalized) ذخیره می‌شود. در نمایش نرمال‌شده، اعداد به صورت `1.xxxxx × 2^exponent` ذخیره می‌شوند، که در آن `x` نشان‌دهنده بیت‌های کسری است.

#### نحوه تبدیل یک عدد حقیقی به نمایش اعشاری

برای تبدیل یک عدد حقیقی به فرمت اعشاری، مراحل زیر انجام می‌شود:

**مثال**: فرض کنید می‌خواهیم عدد 10.25 را در نمایش اعشاری ذخیره کنیم.

**تبدیل عدد به باینری**

   - عدد 10 به باینری تبدیل می‌شود: 

     ```

     10 = 1010

     ```

   - عدد 0.25 به باینری تبدیل می‌شود: 

     ```

     0.25 = 0.01

     ```

   - بنابراین، عدد 10.25 به صورت باینری: 

     ```

     10.25 = 1010.01

     ```

**نرمال‌سازی (Normalization)**

   - عدد باینری را به صورت نرمال‌شده تبدیل می‌کنیم:

     ```

     10.25 = 1.01001 × 2^3

     ```

   - در اینجا، `1.01001` مقدار کسری و `3` نمایه است.

**محاسبه علامت (Sign)**

   - از آنجایی که عدد 10.25 مثبت است، علامت برابر با `0` خواهد بود.

**محاسبه نمایه (Exponent)**

   - برای محاسبه نمایه، ما از یک offset استفاده می‌کنیم. در فرمت IEEE 754 برای اعداد 32 بیتی، offset برابر با 127 است.

   - نمایه `3` را با offset جمع می‌کنیم:

     ```

     3 + 127 = 130

     ```

   - حالا نمایه به باینری تبدیل می‌شود:

     ```

     130 = 10000010

     ```

**محاسبه مقدار کسری (Mantissa)**

   - مقدار کسری که باید ذخیره شود، بیت‌های بعد از `1.` در نرمال‌سازی است.

   - در این مثال، مقدار کسری `01001` است. 

   - از آنجا که فضای ما برای ذخیره مقدار کسری 23 بیت است، ما باید `01001` را با صفرها پر کنیم:

     ```

     01001000000000000000000

     ```

#### ترکیب اجزا

اکنون که همه اجزا را محاسبه کرده‌ایم، می‌توانیم عدد را در فرمت اعشاری ذخیره کنیم:

- **علامت (S)**: `0`

- **نمایه (E)**: `10000010`

- **مقدار کسری (M)**: `01001000000000000000000`

**نتیجه نهایی**

عدد 10.25 در فرمت اعشاری به صورت زیر ذخیره می‌شود:

```

0 | 10000010 | 01001000000000000000000

```

#### نمایش اعداد منفی

برای نمایش اعداد منفی، تنها کافی است علامت S برابر با `1` قرار داده شود. به عنوان مثال، اگر بخواهیم عدد -10.25 را ذخیره کنیم، مراحل همانند قبل خواهد بود، اما در مرحله علامت، S برابر با `1` خواهد بود.

#### دقت و محدوده

محدوده و دقت نمایش اعشاری بستگی به تعداد بیت‌ها در قسمت‌های مختلف دارد. در فرمت‌های مختلف (32 بیتی، 64 بیتی و غیره) می‌توانیم دقت‌های مختلفی را برای نمایش اعداد انتخاب کنیم.
